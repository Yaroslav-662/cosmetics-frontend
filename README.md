# Лабораторна робота №4  
## Реалізація React проєкту з використанням TypeScript та Tailwind CSS

---

## Опис проєкту
Даний проєкт розроблено в межах лабораторної роботи №4 з дисципліни «Веб-програмування».  
Метою проєкту є створення базової архітектури сучасного фронтенд-застосунку з використанням бібліотеки React та мови TypeScript.

Застосунок реалізовано у вигляді Single Page Application (SPA), що забезпечує швидку навігацію між сторінками без перезавантаження браузера. У межах проєкту реалізовано маршрутизацію, переусні компоненти інтерфейсу користувача, інтеграцію з серверним API та адаптивну стилізацію за допомогою Tailwind CSS.

Проєкт є клієнтською частиною веб-системи та слугує основою для подальшого розвитку функціональності фронтенд-рівня.

---

## Технології
У процесі виконання лабораторної роботи було використано такі технології та інструменти:

- **React 18** — бібліотека для побудови користувацьких інтерфейсів
- **TypeScript** — мова програмування з підтримкою статичної типізації
- **Vite** — інструмент для швидкої ініціалізації та збірки фронтенд-проєктів
- **Tailwind CSS** — утилітарний CSS-фреймворк для адаптивної стилізації
- **React Router** — бібліотека для маршрутизації у React-застосунках
- **Axios** — HTTP-клієнт для взаємодії з серверним API
- **ESLint** — інструмент статичного аналізу коду
- **Prettier** — автоматичний форматувальник коду

## Встановлення та запуск

### Вимоги
Для коректної роботи проєкту необхідно попередньо встановити:
- Node.js версії **18 або вище**
- пакетний менеджер **npm** або **yarn**

### Інструкції
Для запуску проєкту необхідно виконати наступні кроки:

```bash
# Клонування репозиторію з GitHub
git clone https://github.com/USERNAME/react-lab-4.git

# Перехід у директорію проєкту
cd react-lab-4

# Встановлення залежностей
npm install

# Створення файлу змінних середовища
cp .env.example .env

# Запуск застосунку у режимі розробки
npm run dev

# Збірка проєкту для продакшену
npm run build
Після запуску сервер розробки буде доступний у браузері за адресою
http://localhost:5173.

Структура проєкту
Проєкт має модульну структуру, яка спрощує підтримку та масштабування коду:

text
Копіювати код
src/
├── components/      # Переусні UI-компоненти
├── pages/           # Компоненти сторінок застосунку
├── services/        # Конфігурація Axios та API-сервіси
├── types/           # TypeScript типи та інтерфейси
├── utils/           # Допоміжні функції
└── hooks/           # Кастомні React hooks
Така організація файлів дозволяє дотримуватися принципів чистої архітектури та забезпечує зручну навігацію у проєкті.
